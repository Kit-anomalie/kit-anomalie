Kit Anomalie — Contenu textuel

Cette page statique est un miroir lisible du contenu du Kit Anomalie pour les outils qui ne savent pas exécuter le JavaScript de la PWA (LLM, crawlers, lecteurs d'écran simplifiés). Le produit réel est une PWA React mobile-first disponible à la racine du domaine.

Identité du produit

Nature. PWA mobile-first installable sur smartphone, fonctionne offline. Compagnon pour agents techniques terrain en maintenance d'infrastructure.

Positionnement. Le kit ne remplace pas les applications métier existantes. Il les rend utilisables sur le terrain (gants, bruit, zones sans réseau, contraintes de sécurité fortes).

Profils utilisateurs. 3 rôles × 5 spécialités. Chaque agent voit le contenu filtré selon son profil :

  • Rôles : Agent terrain / REQ (Responsable Équipe), Ordonnanceur, Référent Patrimoine (RP)
  • Spécialités : Voie, SEG, EALE, CAT, SM

Conventions UI. Bordures arrondies 16-20 px. Mobile-first, max-width 430 px. Touch targets ≥ 44 px (gants). Animations sobres, plafonnées à 400 ms en stagger.

Briques (architecture)

Le kit est organisé en briques, accessibles depuis une home avec tuiles :

  • Brique 0 — Accueil & Profil : configuration du profil agent (rôle + spécialité), home avec tuiles vers chaque brique.
  • Brique 1 — Guides par application : walkthroughs interactifs pas-à-pas qui aident l'agent à utiliser une appli métier.
  • Brique 2 — Fiches mémo réflexes : aide-mémoires courts pour situations récurrentes.
  • Brique 3 — Quiz : QCM thématiques pour acquérir le réflexe.
  • Brique 4 — Catalogue des anomalies : référentiel hiérarchique Catégorie › Type d'actif › Anomalie. 6 catégories.
  • Brique 5 — Assistant IA : recherche locale fuzzy (Fuse.js, offline).
  • Brique 6 — Glossaire métier : sigles et termes avec définitions courtes.
  • Brique 7 — Administration BO : édition des contenus (non exposée à l'agent).

Brique 1 — Guides (1 guide démo)

Guide « Créer une anomalie dans EF3C0 »

Application métier : EF3C0 · Profil cible : agent terrain (rôle REQ), spécialité Voie · Référentiel MT00342

  1. Ouvrir la tournée en cours. Depuis l'écran d'accueil EF3C0, sélectionner sa tournée du jour. Erreurs fréquentes : tournée non synchronisée (icône verte), pull-to-refresh nécessaire.
  2. Sélectionner l'actif concerné. Dans la liste des actifs, choisir celui où le défaut est constaté (recherche par PK ou numéro possible). Champs : Actif (obligatoire), PK ou localisation. Erreurs : confondre actif linéaire (voie) et ponctuel (appareil de voie).
  3. Créer l'anomalie. Bouton « + » ou « Nouvelle anomalie ». Erreurs : ne pas vérifier les anomalies existantes → risque de doublons.
  4. Renseigner la description. Libellé court (obligatoire) + description détaillée + photo (recommandé). Format : quoi (type de défaut), où (composant précis), depuis quand (ancienneté estimée). Erreurs : « Anomalie constatée » n'est pas exploitable.
  5. Choisir le classement. Sélection dans liste déroulante. En cas de doute, privilégier le plus contraignant + ajouter une note. Erreurs : laisser en NC (Non Classé), sur-classer en S/I.
  6. Valider et synchroniser. Vérification du récapitulatif, enregistrement, synchronisation au retour réseau.

Brique 2 — Fiches mémo (4 fiches démo)

Fiche « Classer une anomalie »

Référentiel MT00342 · Mots-clés : classer, classement, S/I, S/DP, A/P

Quoi faire. Attribuer le bon classement réglementaire à une anomalie détectée lors d'une tournée ou visite.

Comment.

  1. Identifier le type de défaut (sécurité immédiate, sécurité différée, maintenance).
  2. Vérifier le référentiel de classement applicable à sa spécialité.
  3. Choisir le classement parmi : S/I (Sécurité / Immédiat), S/DP (Sécurité / Délai Prescrit), A/P (Autres / Prioritaires), A/M (Autres / Maintenance), A/SURV (Autres / Surveillance), A/DET (Autres / à Déterminer).
  4. Renseigner la DLF (Date Limite de Fin) associée.

Erreurs à éviter. Sur-classer S/I → surcharge de la planification d'urgence. Laisser NC → invisible pour l'ordonnanceur. Oublier la DLF → l'anomalie sort du suivi.

Fiche « Rédiger une bonne description »

Mots-clés : description, rédiger, libellé

Quoi faire. Rédiger une description exploitable par le RP et l'Ordonnanceur, sans retour terrain.

Comment. Quoi (type de défaut) · Où exactement (composant + PK + repère, ex : « Joint isolant voie 1, PK 142+350, côté gauche sens croissant ») · Depuis quand · Gravité observée (dimensions, étendue) · Photo cadrée avec repère d'échelle.

Erreurs à éviter. « Anomalie constatée » sans précision. Description trop technique sans localisation. Pas de photo.

Fiche « Éviter les doublons »

Mots-clés : doublon, doublet, existant, vérifier

Quoi faire. Vérifier qu'une anomalie n'existe pas déjà sur l'actif avant d'en créer une nouvelle.

Comment.

  1. Avant de déclarer, vérifier les anomalies déjà embarquées sur l'actif.
  2. Vérifier les anomalies ouvertes (même localisation, même type de défaut).
  3. Si similaire : ne pas recréer, mettre à jour la description si évolution. Si même actif mais défaut différent : créer. Si même défaut mais actif différent (ex : 2 rails distincts) : créer.
  4. En cas de doute, demander au Référent Patrimoine.

Erreurs à éviter. Créer sans vérifier. Mettre à jour une anomalie résolue. Confondre actif linéaire et ponctuel.

Fiche « Comprendre la DLF »

Référentiel MT00342 · Mots-clés : DLF, date limite, délai

Quoi faire. Comprendre et respecter la Date Limite de Fin associée à chaque anomalie classée.

Comment. La DLF dépend du classement : S/I → traitement immédiat · S/DP → délai prescrit · A/P → prioritaire · A/M → prochain cycle de maintenance · A/SURV → réévaluation à date.

Responsabilités. Agent terrain → renseigne le classement, la DLF en découle. Ordonnanceur → planifie avant la DLF. RP → vérifie la cohérence.

Erreurs à éviter. Ignorer la DLF, modifier sans analyse de risque, ne pas renseigner de classement.

Brique 4 — Catalogue des anomalies (6 catégories)

Le catalogue est hiérarchique : Catégorie › Type d'actif › Anomalie. Chaque anomalie contient : description, défaut visible, méthode de mesure (« écart »), matrice de classements selon conditions (avec action recommandée), illustration, référence réglementaire éventuelle.

Catégorie 1 — Voie courante #00A3E0

Rail, joint, traverse, ballast, attaches, soudure, géométrie.

Exemples d'anomalies :

  • VC-RAIL-01 — Usure ondulatoire. Corrugation du champignon. Mesure au gabarit ou Mauzin. Classements : A/DET (≤ 0,3 mm), A/SURV (0,3-0,5 mm), A/P (> 0,5 mm).
  • VC-RAIL-02 — Fissure transversale de rail. Fissure perpendiculaire à l'axe. Contrôle ultrasons. Classements : A/SURV (superficielle non évolutive), S/I (évolutive ou profondeur > 3 mm) → pose immédiate de clips + remplacement sous 48 h.
  • VC-JOINT-01 — Décalage au joint. Dénivelé au droit du joint. Mesure règle + pige. Classements : A/DET (≤ 1 mm), A/M (> 1 mm).
  • VC-TRAV-01 — Traverse dégradée. Bois pourri ou béton fissuré. A/P → remplacement programmé sous 1 mois.

Catégorie 2 — Appareils de voie #0C1E5B

Aiguilles, cœurs, contre-rails, coussinets, traverses d'appareil de voie.

Exemples d'anomalies :

  • Usure de cœur de croisement. Mesure au gabarit. Classements progressifs selon la profondeur.
  • Jeu d'aiguille. Contrôle au calibre. Classements selon le jeu mesuré.
  • Coussinet déformé. Inspection visuelle + clé dynamométrique.

Catégorie 3 — Ouvrages & Gabarit #3AAA35

Ouvrages d'art (ponts, tunnels, murs de soutènement), gabarit cinématique.

Exemples d'anomalies :

  • Fissure d'ouvrage en béton.
  • Corrosion d'éléments métalliques.
  • Empiètement de gabarit (végétation, structure).

Catégorie 4 — Abords #A85D3F

Végétation, propreté des emprises, accès, clôtures.

Exemples d'anomalies :

  • Végétation envahissante sur emprise.
  • Clôture endommagée.
  • Accès non sécurisé.

Catégorie 5 — Installations signalisation #F7A600

Feux, panneaux, balises, dispositifs de signalisation au sol.

Exemples d'anomalies :

  • Feu défaillant (LED grillée, alimentation coupée).
  • Panneau dégradé ou illisible.
  • Balise déplacée ou manquante.

Catégorie 6 — Platelage #475569

Passages à niveau, planches de platelage, fixations.

Exemples d'anomalies :

  • Planche cassée ou décalée.
  • Écart entre planches dépassant la tolérance.
  • Fixation lâche ou manquante.

Brique 3 — Quiz

QCM thématiques avec score et historique. Thèmes couverts : classements, description d'anomalie, gestion des doublons, DLF. Format : question + 4 réponses cliquables, feedback immédiat à chaque réponse, score final + recommandations de fiches mémo à revoir si erreurs.

Brique 5 — Assistant IA

Recherche fuzzy locale (Fuse.js) sur l'ensemble du contenu (guides, fiches, catalogue, glossaire). Aucun appel serveur — tout est indexé côté client. Résultats groupés par type, tri par pertinence. L'agent tape un mot-clé, voit immédiatement les ressources les plus pertinentes de son profil.

Caractéristiques techniques

  • PWA installable (Service Worker, manifest.json)
  • Auto-healing : purge des caches au changement de version, sync auto du contenu via content.json
  • Workflow contenu : un éditeur principal + contributeurs locaux (admin form-based, exports JSON, fusion sans écrasement)
  • Stack React 19 + TypeScript + Vite + Tailwind 4 + Zustand 5 + React Router 7
  • Hébergement actuel : GitHub Pages (statique)
  • Migration future : SI interne (GitLab + PostgreSQL) une fois adopté à plus grande échelle

Tonalité du produit

Voix. Direct, factuel, terrain. Pas de jargon corporate. Phrases courtes. On tutoie l'agent quand on s'adresse à lui.

Conventions de wording. « DLF » et non « date de fiabilisation ». « Anomalie » et non « défaut » (un défaut peut être un constat sans anomalie). « Embarquement » pour l'assignation automatique aux tournées.

Limites assumées. Le kit n'a pas vocation à remplacer les applis métier. Il accompagne, il ne saisit pas pour l'agent.
